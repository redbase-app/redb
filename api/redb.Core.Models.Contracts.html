Namespace redb.Core.Models.Contracts

Interfaces

IRedbList

REDB list interface Represents a reference list for list-type fields

IRedbListItem

REDB list item interface Represents a reference list element

IRedbObject

Base interface for all REDB objects Allows working with objects as classes rather than IDs Enriched with full functionality for working with trees, timestamps and auditing

IRedbObject<TProps>

Typed interface for REDB objects with specific properties type Extends base IRedbObject adding type-safe access to properties Provides IntelliSense and compile-time type checking

IRedbPermission

REDB permission interface Represents user or role access rights to object/schema

IRedbRole

REDB role interface Represents user role in security system

IRedbScheme

REDB scheme interface Represents scheme (type) of objects in system with structure encapsulation

IRedbSecurityContext

REDB security context Manages current user and operation mode

IRedbStructure

REDB scheme field structure interface Represents field metadata in object scheme

IRedbType

REDB data type interface Represents data type for fields in schemes

IRedbUser

REDB user interface Represents system user with his basic properties

IRedbUserRole

REDB user-role relationship interface Represents role assignment to a user

ISchemeCacheProvider

Extension of ISchemeSyncProvider with metadata cache management methods Adds capabilities for hot switching, invalidation and cache warmup

ITreeNode<T>

Interface for tree nodes with navigational properties and traversal operations

ITreeRedbObject

Interface for REDB tree objects with navigational properties Extends IRedbObject adding tree navigation capabilities in memory Supports polymorphic trees (objects of different schemes in one tree)

ITreeRedbObject<TProps>

Typed interface for tree objects with specific property type. Combines ITreeRedbObject (untyped navigation) and IRedbObject<TProps> (typed data). Supports polymorphic trees - parents and children can be different types.
