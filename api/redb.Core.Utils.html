Namespace redb.Core.Utils

Classes

DateTimeConverter

Centralized DateTime conversion logic. RULE: DateTime is ALWAYS treated as UTC in REDB system.

CONTRACT:

  • On save: any DateTime → UTC (via NormalizeForStorage)
  • On read: DateTimeOffset from DB → DateTime with Kind=Utc (via DenormalizeFromStorage)
  • On search: any DateTime → UTC (via NormalizeForStorage)
  • On JSON: DateTimeOffset → DateTime with Kind=Utc (via DenormalizeFromStorage)

IMPORTANT: DateTimeOffset remains as is (preserves timezone information)

RedbHash

Utility for computing MD5 hash from object properties. Supports both generic RedbObject{TProps} (hash from Props) and non-generic RedbObject (hash from base value_* fields).

RedbKeySerializer

Serializes and deserializes Dictionary keys to/from _array_index (text) storage. Simple keys (string, int, Guid, etc.) are serialized directly. Complex keys (Tuple, custom classes) are serialized to Base64-encoded JSON.

RedbTypeIds

System type IDs for REDB. Matches _types table in database. Negative IDs are reserved for system types.

RedbTypeMapping

Helper methods for REDB type mapping between C# types and database columns

SchemeHashCalculator

Utility for computing MD5 hash of scheme structure. Used for change detection and cache invalidation.

TreeExtensions

Extensions for working with tree structures.

TreeObjectConverter

Utility class for converting between RedbObject and TreeRedbObject types. Provides static methods for type conversion and parent chain building.

ValuesTopologicalSort

Topological sorting of _values to maintain FK constraint on _array_parent_id. Parent records must be inserted BEFORE children in BulkInsert.
