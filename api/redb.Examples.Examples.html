E000_BulkInsert

Example: Bulk insert of complex employee records.

Classic ORM (EF Core) would require ~28 tables:

  1. Employees - main table
  2. EmployeeSkills - string[] Skills (FK)
  3. EmployeeSkillLevels - int[] SkillLevels (FK)
  4. EmployeeCertifications - string[] (FK)
  5. EmployeeCertificationYears - int[] (FK)
  6. Addresses - for Home/Work/Emergency (FK)
  7. BuildingInfos - nested in Address (FK)
  8. BuildingAmenities - string[] in BuildingInfo (FK)
  9. BuildingAccessCodes - int[] in BuildingInfo (FK)
  10. BuildingParkingSpots - string[] (FK)
  11. BuildingElevatorFloors - int[] (FK)
  12. Contacts - Contact[] (FK)
  13. ContactNotificationHours - int[] in Contact (FK)
  14. ContactMetadata - ContactDetail[] in Contact (FK)
  15. ProjectMetrics - RedbObject reference
  16. ProjectTags - Tag[] in ProjectMetrics (FK)
  17. ProjectTechnologies - string[] in ProjectMetrics (FK)
  18. EmployeePastProjects - many-to-many (2 FK)
  19. PhoneDirectory - Dictionary (FK)
  20. OfficeLocations - Dictionary + Address (2 FK)
  21. BonusByYear - Dictionary (FK)
  22. DepartmentHistory - Dictionary (FK)
  23. Departments - nested class
  24. DepartmentProjects - string[] (FK)
  25. DepartmentLeaders - Tag[] (FK)
  26. DepartmentBudgetByYear - nested Dictionary (FK)
  27. PerformanceReviews - tuple key Dictionary (FK)
  28. ProjectMetricsDict - Dictionary with RedbObject (2 FK)

INSERTs per employee: ~40-60 records across 28 tables with FK ordering.

For 100 employees: 4000-6000 INSERTs!

REDB: 2 tables (_objects + _values), ~3000 records, single BulkInsert via COPY.

E088_TreeCreateBulk

BULK create tree hierarchy (Pro feature) - FAST version.

Creates ~100 tree nodes using pre-generated IDs + AddNewObjectsAsync. Compare with E089 (sequential CreateChildAsync).

Structure:

  • 1 root (TechCorp)
  • 10 regional offices
  • 50 departments (5 per office)
  • 40 teams (4 per first 10 departments)

Optimization:

  • NextObjectIdBatchAsync - get all IDs in 1 query
  • AddNewObjectsAsync - bulk insert all nodes in 1 operation
E089_TreeCreate

Sequential create tree hierarchy (Pro feature) - SLOW version.

Creates ~100 tree nodes using sequential CreateChildAsync. Compare with E088 (bulk AddNewObjectsAsync).

Structure:

  • 40 teams (4 per first 10 departments)

Each CreateChildAsync = separate DB round-trip.
