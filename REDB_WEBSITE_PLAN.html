REDB.Doc.Web - Website Implementation Plan

Overview

Full-featured website for REDB framework: marketing, documentation, user accounts, and licensing.

Technology Stack:

  • Backend: ASP.NET Core 9.0 (Blazor Server)
  • UI: Fluent UI Components
  • Database: REDB (PostgreSQL)
  • Payment: Stripe
  • Email: SendGrid

Key Decisions

1. Authentication

Use custom authentication stored in REDB (not ASP.NET Identity with separate DB):

  • UserAccountProps in REDB
  • Custom AuthenticationStateProvider
  • JWT tokens for API access

2. Payment

Stripe with:

  • Checkout Sessions (redirect to Stripe)
  • Customer Portal (manage subscriptions)
  • Webhooks (subscription events)

3. License Keys

JWT-based licenses:

  • Generated on subscription creation
  • Signed with private key
  • Validated in NuGet packages with public key
  • Revocation support via database check

4. Search

REDB Query API:

  • Full-text search on all Props fields
  • No external search engine needed
  • Fast with proper indexing

5. Content Management

REDB-based CMS:

  • All content in REDB (products, pages, examples)
  • Admin panel for editing
  • No external CMS
